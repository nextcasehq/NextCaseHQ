# Pending Integration Register

This register tracks deferred integration, testing, and optimization work for NextCaseHQ Modules 13–20.

## Resolved: Former Critical Production Blocker (Admin Panel Authentication)

**Resolved by PR #65 (commit `4b41b98`).** The admin console's authentication used to be entirely client-side (`apps/web/src/app/admin/page.tsx` checked a hardcoded password — one valid value was literally the word `admin` — and wrote a hardcoded `NEXTCASE_ADMIN_TOKEN` cookie value itself). It is now server-verified: `POST /api/admin/session` checks the submitted key against `ADMIN_ACCESS_TOKEN` (constant-time, rate-limited, origin-checked) and mints a signed, httpOnly, Secure, SameSite=Strict session cookie; `middleware.ts`'s `/api/admin/*` gate verifies that signed token. The admin UI's markup/labels/tabs are unchanged — only the login/logout/session-check wiring changed. See `known_limitations.admin_auth_shared_secret_no_individual_accountability` in `docs/BUILD_LEDGER.json` for the one tradeoff this milestone deliberately left open (a single shared operator secret, not per-admin accounts).
